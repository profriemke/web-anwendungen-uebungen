Capstone Übung C.3.4: Die Rechnung zum Selbstdruck

Lösung

Lösungsidee

Ein wichtiger Baustein ist die Beachtung des Toolings. Danach nehmen wir uns die Problempunkte einzeln vor und überschreiben nicht sinnvolle Werte in einem @media print Media Query.

Für den Druck müssen folgende Punkte angepasst werden:

  • Seitenformat
  • Typografie und Grundlayout
  • Rechnungscontainer an A4 angepassen
  • Nur für Bildschirm relevante Elemente ausblenden
  • Farbliche Hervorhebungen druckfreundlich machen
  • Optional: Seitenumbrüche optimiert

Umsetzung

  • Seitenformat
    • Einführung eines @page-Blocks mit Seitenrändern:
      @page { margin: 15mm 18mm; }
  • Typografie und Grundlayout
    • Anpassung der Basis-Schriftgröße für den Druck:
      body { font-size: 11pt; }
    • body-Hintergrund auf Weiß gestellt und Standardabstände entfernt:
      body { background: #ffffff; margin: 0; }
    • Außenabstand der Wrapper-Seite reduziert:
      .page { padding: 0; }
  • Rechnungscontainer an A4 angepasst
    • Die Karte wird zur vollen Seitenbreite „entkapselt“:
      • max-width: 100%;
      • margin: 0;
      • box-shadow: none;
      • border-radius: 0;
      • border: none;
  • Nur für Bildschirm relevante Elemente ausblenden
    • Navigation, Toolbar, Header und Hinweistext über .screen-only ausgeblendet:
      .screen-only { display: none; }
    • Buttons und Toolbar-Actions im Ausdruck entfernt:
      • .button { display: none; }
      • .toolbar-actions { display: none; }
    • Hinweis unter der Rechnung im Druck nicht anzeigen:
      .hint { display: none; }
  • Farbliche Hervorhebungen druckfreundlich gemacht
    • Badges von farbigen Flächen zu dezenten Rahmen-Badges geändert:
      • background: transparent;
      • color: #000000;
      • border: 1px solid #000000;
    • Abschnittstitel in Schwarz gesetzt:
      .section-title { color: #000000; }
  • Seitenumbrüche optimiert
    • Verhindern von Seitenumbrüchen innerhalb wichtiger Blöcke:
      • .items table { page-break-inside: avoid; }
      • .footer { page-break-inside: avoid; }
      • .totals { page-break-inside: avoid; }
